Коллизия в HashMap - это ситуация, когда два или более ключевых объекта выдают одно и то же окончательное хеш-значение и, следовательно, указывают на одно и то же местоположение корзины или индекс массива.
Этот сценарий может произойти, потому что два неравных объекта в Java могут иметь одинаковый хеш-код. Это также может произойти из-за конечного размера базового массива, то есть до изменения размера.
Класс HashMap реализован с использованием метода цепочек, т.е. каждой корзине соответствует свой связанный список и при возникновении коллизии осуществляется добавление нового элемента в этот список.
Когда в одной корзине имеется не менее 8 записей (TREEIFY_THRESHOLD), а общее количество корзин - более 64 (MIN_TREEIFY_CAPACITY), то связанный список в этой корзине будет преобразована в древовидную структуру (красно-черное дерево). Если количество элементов в одной корзине уменьшится до 6 (UNTREEIFY_THRESHOLD), то произойдет обратный переход от дерева к связному списку.

Для разрешения коллизий также существует метод открытой адресации, суть которого состоит в том, чтобы, пользуясь каким-либо алгоритмом, обеспечивающим перебор элементов таблицы, просматривать их в поисках свободного места для новой записи.
